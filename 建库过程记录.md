# 建库过程记录

## 目标
从零构建两层记忆库（动态层 + 静态层），支持向量相似度检索。

## 核心设计决策

### 1. 文件结构（重要）
```
outputs/memory/videomme_L_27B/
  └── {video_id}/
      ├── metadata.json          (向量索引，用于快速检索)
      ├── evidence.json          (所有chunks的详细信息)
      └── chunk{id:03d}/
          ├── keyframes/         (3个代表帧)
          │   ├── frame_001.jpg
          │   ├── frame_002.jpg
          │   └── frame_003.jpg
          └── intermediate/      (中间产物)
              ├── captions.json  (帧级描述)
              └── audit.json     (质量评估)
```

**为什么这样设计：**
- metadata.json 轻量级，支持快速向量检索（O(1)加载所有向量）
- evidence.json 包含所有核心信息，大多数查询只需看这个文件
- keyframes/intermediate 分离，减少频繁加载的文件大小
- 按chunk分目录，便于管理和扩展

### 2. 两层信息分离（重要）
- **动态层（narrative layer）**：memory_text, key_events, actors, state_changes, temporal_relations, causal_clues, v_dynamic
- **静态层（visual attributes layer）**：static_attributes, static_index_text, v_static
- 两层都保存在 evidence.json 中，可独立查询

### 3. 向量编码（关键）
- **v_dynamic**：BGE-M3 对 summary + speech_text 的编码（1024维，L2归一化）
- **v_static**：BGE-M3 对 static_index_text 的编码（1024维，L2归一化）
- 两个向量独立，支持不同查询场景

### 4. 帧处理策略
- **采样**：1fps（可配）
- **分组**：SigLIP余弦相似度 + 动态分组（theta=0.80, n_max=30）
- **保存**：每个chunk保存3个代表帧（first, middle, last）
- **描述**：对所有采样帧生成 caption（一帧一个描述）

### 5. 质量过滤（关键改进）
跳过以下低质量 chunk：
- 时长 < 1 秒
- 没有有效 caption（全是"[frame]"）
- memory_text 过短（< 10字符）

**为什么重要：**
- 避免黑帧、转场、无效帧被保存
- 提升检索结果质量
- 减少存储开销

## 检索流程

1. **加载 metadata.json** → 获取所有 chunk 的 v_dynamic, v_static
2. **向量相似度计算** → 找到top-k相关chunks
3. **加载对应的 evidence.json**[k] → 获取详细信息
4. **按需加载** → keyframes, captions 等补充信息

**时间复杂度**：O(1)加载向量 + O(n)相似度计算 + O(1)加载evidence

## 技术栈

- **VLM**：Qwen3.5-27B（vLLM api_server, port 8001）
- **视觉编码**：SigLIP（448x448）
- **语义编码**：BGE-M3（1024维）
- **视频采样**：decord（1fps）
- **数据模型**：Pydantic（MemoryBank, MemoryChunk, StaticAttribute）

## 多端口负载均衡（可选）

支持 `--vlm-api-url http://127.0.0.1:8000,http://127.0.0.1:8001` 格式，
VLMClient 会自动轮询调用不同端口。

## 启动 vLLM

```bash
bash scripts/start_vllm_qwen35.sh [GPU_ID] [PORT]
# 例：
bash scripts/start_vllm_qwen35.sh 1 8001
```

## 启动建库

单粒度（相似度分组）：

```bash
PYTHONPATH=. python -m src.build_memory.build_single_similarity \
  --benchmark videomme \
  --siglip-model /path/to/siglip \
  --api-url http://127.0.0.1:8001 \
  --api-model Qwen3.5-27B \
  --siglip-gpu cuda:6 \
  --bge-gpu cuda:6
```

多粒度（固定时间金字塔 L1=10s / L2=30s / L3/L4=…）：

```bash
PYTHONPATH=. python -m src.build_memory.build_multi_pyramid --help
```

## 关键参数说明

- `--fps`：帧采样率（1.0 = 1fps）
- `--theta`：SigLIP分组余弦相似度阈值（0.80）
- `--n-max`：每个group最多保留的帧数（30）
- `--api-url`：vLLM API地址（支持多端口逗号分隔）
- `--api-model`：vLLM 暴露的模型名

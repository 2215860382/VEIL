# 记忆库构建方法笔记

---

## 1. 预处理

- **抽帧**：ffmpeg 以 1 fps 从视频中抽帧
- **过滤**：
  - 黑帧/纯色帧：先用文件大小快速预筛（JPEG 极小说明像素几乎全同），再用像素均值和标准差精确判断（捕捉接近黑或接近白的帧）
  - 模糊帧：Laplacian 方差低于阈值则丢弃；若一个窗口内 > 50% 帧被判为模糊（低光/动画/PPT 场景），则跳过该窗口的模糊过滤（failsafe）

---

## 2. 帧分组

### 方法一：单粒度相似度分组

- **帧编码**：用 SigLIP 对每帧做 image encoder，得到 L2 归一化向量
  - 同一份模型在 query 时还会用其 text encoder 编码视觉查询 → 这是双路检索的视觉路径
- **相邻贪心分组**：相邻帧余弦相似度高于阈值（0.80）则合并到同一 chunk
- **碎片合并**：1–2 帧的小组折回左邻居（处理切镜/过渡边界帧）
- **组内去重**：
  - dHash（感知哈希，Hamming 距离 ≤ 4）
  - SigLIP 余弦相似度（≥ 0.95）
- Chunk 边界由视觉内容决定，长度可变

### 方法二：多粒度固定分组

- **分组**：按固定时间窗口切分，形成 4 层金字塔
  - L1 = 10s，L2 = 30s，L3 = 3min，L4 = 10min
- **L1 组内去重**：SigLIP 余弦相似度（≥ 0.95），每组保留 Laplacian 方差最大的帧（最清晰）
- L2–L4 不涉及视觉处理，仅做文本聚合

---

## 3. 字幕对齐（若存在 SRT）

- **解析 SRT**：剥掉 HTML 标签，输出 (start, end, text) 三元组列表
- **按 chunk 时间窗切片**：
  - 收集所有与该窗口相交的字幕条目
  - 拼成单段 speech 文本（超过 500 字符按词截断）

---

## 4. 叙述层构建

### 方法一

- **逐帧 caption**：每帧一句短描述（≤ 25 词），要求只描述可见内容，不推测
- **文本聚合**：
  - 输入该 chunk 的逐帧 caption（> 15 帧时等间距采样）+ 对齐的 speech
  - VLM 输出 JSON 格式（summary / key_events / actors / state_changes / temporal_relations / causal_clues）
  - JSON 解析失败回退：取首句 caption
- **文本向量**：用 BGE-M3 对（summary + speech）拼接文本编码，得到语义向量

### 方法二

- **L1**：代表帧（去重后 2–3 张）+ 对齐字幕一起输入 VLM，直接输出纯文本叙述段落（≤ 200 词）
  - 帧按时间顺序传入，prompt 提供各帧时间戳；但 10s 内帧间视觉变化小，VLM 主要依赖字幕理解时序，图像更多补充场景外观
- **L2–L4（纯 LLM，无视觉）**：
  - L2：聚合 3 个相邻 L1（30s 窗口）→ JSON {text, timeline, causality}（≤ 300 词）
  - L3：聚合 6 个相邻 L2（3min 窗口）→ JSON（≤ 1500 词）
  - L4：聚合约 3 个相邻 L3（10min 窗口）→ JSON（≤ 3000 词）
  - 每层要求保留具体数字、人名、OCR 文本，不得捏造
- **文本向量**：用 BGE-M3 对 L1 叙述文本编码；L2–L4 各层独立编码

---

## 6. 视觉层构建

### 方法一

- **关键帧选择**：每个 chunk 内取 Laplacian 方差最大的帧（构建时选定并写入 metadata，推理时也支持动态重算）
- **视觉向量**：复用步骤 2 中该帧的 SigLIP 向量，存入 chunk 的 `v_visual` 字段

### 方法二

- **关键帧选择**：L1 去重后每组已保留最清晰帧，直接使用
- **视觉向量**：SigLIP 向量按 L1 窗口对齐存入 `L1_visual.npz`；L2–L4 无视觉向量
